静态代码扫描工具：Infer

发表于 2015-10-13   |   分类于 持续集成   |  

Infer

MACOS X上安装

  1. 前置条件
    JAVA <=1.7
    安卓开发环境（非必须）
    SDK(http://tools.android-studio.org/index.php/sdk)
    ANDROID STUDIO(http://www.android-studio.org/)

    xcode在6.1~6.3之间
    clang(使用xcode命令行工具,使用命令xcode-select –install 来安装)

  1. 下载代码
    git clone https://github.com/facebook/infer.git

  2. 安装 opam 及依赖
    brew install opam
    opam init –comp=4.01.0
    opam install sawja.1.5 atdgen.1.5.0 javalib.2.3 extlib.1.5.4

  3. 编译
    cd infer
    ./update-fcp.sh && ../facebook-clang-plugin/clang/setup.sh && ./compile-fcp.sh # 时间好长啊
    make -C infer
    make -C infer java # 不需要支持C/Object-C
    export PATH=pwd/infer/bin:$PATH

分析文件或者项目

  1. OC文件
    infer – clang -c <文件名>
    例子：infer – clang -c <文件名>

  2. C文件
    infer – gcc -c <文件名>
    例子：infer – gcc -c hello.c

  3. IOS项目
    infer – xcodebuild -target <项目名> -configuration Debug -sdk iphonesimulator
    例子：infer – xcodebuild -target HelloWorldApp -configuration Debug -sdk iphonesimulator
    Build settings from command line:
    如果需要再次分析，需要先CLEAN
    xcodebuild -target HelloWorldApp -configuration Debug -sdk iphonesimulator clean
    infer – xcodebuild -target HelloWorldApp -configuration Debug -sdk iphonesimulator

  4. C项目
    infer – make
    再次分析 infer -i – make 或者在infer – make执行前执行make clean

  5. JAVA文件
    infer – javac java文件名
    例子： infer – javac Hello.java

  6. 安卓项目
    在工程目录下面放一个文件 local.properties,内容为 sdk.dir=/my_current_way_to/sdk
    infer – ./gradlew build

infer执行的两个阶段

  1. 转化阶段
    infer将原本各个构建系统自带的构建命令包含其中，目的是用infer将其编译的过程转化为infer自带的语言翻译过来。转换后的东西默认是统计目录下的infer-out目录，但是可以通过-O参数修改。
  2. 分析阶段
    infer从infer-out目录中分析第一阶段编译的结果。它会分析每一个函数，一担发现某个函数有错误，infer会停在这个方法有错误的语句上面。会跳过这个函数继续向下执行分析其他函数。所以infer工作流不仅仅为infer运行code阶段定义，也有可能在修复错误阶段，发现更多错误的阶段以及判断所有错误是否都被修复了这个阶段，这些过程中都会有infer工作流程所定义的过程。
  3. 增量和费增量工作流的含义
    我们知道，我们每一次的分析过程都会生成infer-out目录，对于这个目录，如果下次执行infer的时候，都要先删除这个目录，重新生成，我们称这样的工作流叫非增量工作流。但是如果我们要保留infer-out目录，每次只分析增加的新文件的信息，可以通过参数-i(或—-incremental)来改变，这样我们的工作流就称为增量工作流。

infer-debug模式

–debug或–frontend_debug –debug
例如我们用debug模式分析iOS项目:
infer –debug -i – xcodebuild -target HelloWorldApp -configuration Debug -sdk iphonesimulator

输出的过程会有更多更详细的系数打印出来，且在结果目录下会生成更多的文件，多出来一些.dot和.html后缀文件

捕捉的BUG类型

Java中捕捉的bug类型

Resource leak
Null dereference
C/OC中捕捉的bug类型

Resource leak
Memory leak
Null dereference
Premature nil termination argument
只在 OC中捕捉的bug类型

Retain cycle
Parameter not null checked
Ivar not null checked
