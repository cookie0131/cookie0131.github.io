分库分表的集中常见形式：垂直分库

发表于 2015-06-13   |   分类于 架构   |  

垂直分库的思路就是按照业务模块来划分出不同的数据库表，二不是像早期一样将所有的数据都放在同一个数据库中。比如把一个DB库拆分成 User库，Order库，Product库

系统层面的服务化拆分操作，能够解决业务系统层面的耦合和性能瓶颈，有利于系统的扩展维护。二数据库层面的拆分，道理也是相同的。与服务的治理和降级机制类似，我们也能对不同业务类型的数据进行分级管理，维护，监控，扩展等

数据库往往容易成为应用系统的瓶颈，而数据库本身属于有状态的，相对于web应用服务器，比较难实现横向扩展。数据库的连接资源比较宝贵且单机处理能力也有限，在高并发场景下垂直分库一定程度上能够突破Io,连接数及单机硬件资源的瓶颈，是大型分布式系统中优化数据库架构的重要手段

分库分表的集中常见形式：垂直分表

发表于 2015-06-01   |   分类于 架构   |  

垂直分表通俗的说法是大表拆小表，拆分是基于关系型数据库中的列进行的。通常某个表的字段比较多，可以新建立一张扩展表，将不经常使用或者长度较大的字段拆分出去放到扩展表中

原表:article

id title createtime author content

拆分为：article_content

id content articleid

和 article_info

id title createtime author

拆分字段的操作建议在数据库设计阶段就做最好，如果是在发展过程中拆分，组需要改写以前的查询语句，会额外带来一定的成本和风险

Monkey测试介绍

Monkey测试是Android平台自动化测试的一种手段，通过Monkey程序模拟用户触摸屏幕、滑动Trackball、按键等操作来对设备上的程序进行压力测试，检测程序多久的时间会发生异常。

Monkey测试特点

Monkey测试的特点主要有以下几点：

  • 可对MonkeyTest的对象，事件数量，类型，频率等进行设置。
  • Monky测试使用的事件流数据流是随机的，不能进行自定义。
  • 测试的对象仅为应用程序包，有一定的局限性。
